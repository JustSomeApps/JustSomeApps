Salesforce Architecture and Solution Design

Salesforce can be a hairy beast of a toolset. It’s important to have the right architect to define the right plan. At Just Some Apps, we have tried and true experience architecting the right solution for your business needs and goals.

Let’s not make it complicated! We focus on the following four pillars of Architecture and Design:

  1. Design for the future process, not the current state (Read: LET GO of the baggage!)
  2. Code when needed, not just because it’s cooler (it’s totally cooler, we get it)
  3. Don’t configure Frankenstein, use code if it’s more elegant (this is for those admins out there, just because you can build a 20 step process flow with all the bells and whistles…should you?)
  4. Architect a system that we’d want to use (in other words, let’s not spend a lot of money on something that someone’s spreadsheet will outlive)
